# 283.移动零
给定一个数组 nums，编写一个函数将所有 0 移动到数组的末尾，同时保持非零元素的相对顺序。请注意 ，必须在不复制数组的情况下原地对数组进行操作。\
这是一个经典的双指针（Two Pointers）原地修改数组问题。最优解法是使用双指针法（快慢指针交换）
## 算法思路（快慢双指针）
1.初始化双指针：\
&emsp;&emsp;慢指针 slow：指向“下一个非零元素应该存放的位置”。初始为 0。\
&emsp;&emsp;快指针 fast：用于遍历整个数组，寻找非零元素。初始为 0。\
2.遍历与交换：\
&emsp;&emsp;快指针 fast 向后移动。\
&emsp;&emsp;每当 nums[fast] 不等于 0 时：\
&emsp;&emsp;将 nums[fast] 与 nums[slow] 的值进行交换。\
&emsp;&emsp;交换后，慢指针 slow 向后移动一位。\
3.逻辑结果：\
&emsp;&emsp;交换动作确保了非零元素被依次“滚”到了数组的前部。\
&emsp;&emsp;由于慢指针只有在遇到非零数并完成交换后才前进，因此慢指针左侧永远是非零数，而快慢指针之间沉淀下来的全都是零。
## pyton3代码实现
```python
from typing import List

class Solution:
    def moveZeroes(self, nums: List[int]) -> None:
        """
        Do not return anything, modify nums in-place instead.
        """
        # 1. 初始化慢指针，代表当前处理完的非零序列的末尾
        slow = 0
        
        # 2. 快指针 fast 遍历整个数组
        for fast in range(len(nums)):
            # 3. 发现非零元素
            if nums[fast] != 0:
                # 4. 交换快慢指针指向的值
                # 如果快慢指针相等（即前面没有0），交换操作相当于原地不动
                nums[slow], nums[fast] = nums[fast], nums[slow]
                
                # 5. 慢指针前进一步
                slow += 1

# --- 测试代码 ---
if __name__ == "__main__":
    sol = Solution()
    
    # 测试用例 1
    example1 = [0, 1, 0, 3, 12]
    sol.moveZeroes(example1)
    print(f"输出 1: {example1}") # 预期: [1, 3, 12, 0, 0]
    
    # 测试用例 2
    example2 = [0]
    sol.moveZeroes(example2)
    print(f"输出 2: {example2}") # 预期: [0]
```
## 逻辑图解（以 [0, 1, 0, 3, 12] 为例）
1.初始：slow=0, fast=0。nums[0]是0，跳过。\
2.步骤1：fast=1。nums[1]=1（非零）。\
&emsp;&emsp;交换 nums[0] 和 nums[1]。数组变为 [1, 0, 0, 3, 12]。\
&emsp;&emsp;slow 变为 1。\
3.步骤2：fast=2。nums[2]是0，跳过。\
4.步骤3：fast=3。nums[3]=3（非零）。\
&emsp;&emsp;交换 nums[1] 和 nums[3]。数组变为 [1, 3, 0, 0, 12]。\
&emsp;&emsp;slow 变为 2。\
5.步骤4：fast=4。nums[4]=12（非零）。\
&emsp;&emsp;交换 nums[2] 和 nums[4]。数组变为 [1, 3, 12, 0, 0]。\
&emsp;&emsp;slow 变为 3。\
6.结束：所有非零数已保持顺序移至前方。
## 复杂度分析
时间复杂度: $O(n)$\
&emsp;&emsp;只需对数组进行一次线性遍历。快指针 fast 从头走到尾。\
空间复杂度: $O(1)$\
&emsp;&emsp;仅使用了 slow 和 fast 两个辅助变量，直接在原数组上进行修改，符合题目“原地操作”的要求。

# 11.盛最多水的容器
给定一个长度为 n 的整数数组 height 。有 n 条垂线，第 i 条线的两个端点是 (i, 0) 和 (i, height[i]) 。找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。返回容器可以储存的最大水量。说明：你不能倾斜容器。这是一个经典的双指针（Two Pointers）贪心策略问题。问题的本质是：通过排除法缩小搜索空间。当我们计算当前左右边界构成的面积后，为了寻找潜在更大的面积，我们必须移动其中一个指针。如果移动较高的那根木板，容器的限制高度（短板）不会增加，而宽度却在减小，面积必然变小。因此，唯一可能让面积增大的操作是移动那根较短的木板，寄希望于在内部找到一根更长的木板。\
最优解法是使用双指针法（左右边界向中间收缩）
## 算法思路（对撞双指针）
1.初始化双指针：\
&emsp;&emsp;左指针 left 指向数组起始位置（0）。\
&emsp;&emsp;右指针 right 指向数组末尾位置（n-1）。\
2.循环计算面积：\
&emsp;&emsp;当 left < right 时，重复以下步骤：\
&emsp;&emsp;计算当前面积： 面积 $=\min ($ height $[$ left $]$, height $[$ right $]) \times($ right - left $)$ 。\
&emsp;&emsp;更新最大面积：对比当前面积与历史最大面积 max_area。\
3.贪心移动指针：\
&emsp;&emsp;比较 height[left] 和 height[right]：\
&emsp;&emsp;如果左侧木板较短，则 left += 1。\
&emsp;&emsp;如果右侧木板较短，则 right -= 1。\
4.返回结果：当指针相遇，扫描结束，返回 max_area。
## python3代码实现
```python
from typing import List
class Solution:
    def maxArea(self, height: List[int]) -> int:
        # 1. 初始化左右指针
        left = 0
        right = len(height) - 1
        max_area = 0
        
        # 2. 当左右指针未相遇时执行循环
        while left < right:
            # 3. 计算当前容器的高度（取短板）
            current_height = min(height[left], height[right])
            # 4. 计算当前容器的宽度
            current_width = right - left
            
            # 5. 计算面积并更新全局最大值
            current_area = current_height * current_width
            max_area = max(max_area, current_area)
            
            # 6. 核心贪心策略：移动较短的那根木板
            if height[left] < height[right]:
                left += 1
            else:
                right -= 1
                
        return max_area

# --- 测试代码 ---
if __name__ == "__main__":
    sol = Solution()
    
    # 测试用例 1
    case1 = [1, 8, 6, 2, 5, 4, 8, 3, 7]
    print(f"输入 1: {case1} -> 输出: {sol.maxArea(case1)}") # 预期: 49
    
    # 测试用例 2
    case2 = [1, 1]
    print(f"输入 2: {case2} -> 输出: {sol.maxArea(case2)}") # 预期: 1
```
## 逻辑图解（以 [1, 8, 6, 2, 5, 4, 8, 3, 7] 为例）
1.初始：left=0 (h=1), right=8 (h=7)。宽度 8，面积 1×8=8。\
2.移动：左侧短，left 右移。\
3.下步：left=1 (h=8), right=8 (h=7)。宽度 7，面积 7×7=49。max_area = 49。\
4.下步：右侧短，right 左移。\
5....持续收缩... 只有当 left 停在 8，right 停在 8 那个位置（索引 6）时，面积最大。
## 复杂度分析
时间复杂度：O(n)\
&emsp;&emsp;双指针从两端向中间移动，数组中的每个元素只被访问一次，总步数为 n。\
空间复杂度：O(1)\
&emsp;&emsp;只使用了常数个变量存储指针位置和面积最大值，不需要额外的存储空间。
# 15.三数之和
给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0 。请你返回所有和为 0 且不重复的三元组。注意：答案中不可以包含重复的三元组。这是一个经典的“排序 + 双指针”搜索问题。问题的本质是：将三数之和转化为“定一动二”的二数之和。我们可以固定一个数 a，然后在剩下的数组中寻找两个数 b和 c，使得 b+c=−a。通过先对数组进行排序，我们可以利用双指针在 $O(n)$ 的时间内扫完剩余部分，并且通过跳过相同的数字来完美解决去重问题。最优解法是使用 排序 + 左右双指针法。
## 算法思路（排序 + 对撞指针）
1.预处理：先将数组 nums 进行升序排序。\
2.固定首位：遍历排序后的数组，以 nums[i] 作为三元组中的第一个元素。\
&emsp;&emsp;去重剪枝：如果 i > 0 且 nums[i] == nums[i-1]，直接跳过，因为以该值开头的组合已经找过了。\
&emsp;&emsp;提前终止：如果 nums[i] > 0，由于数组是升序的，后面的数也一定大于 0，三数之和不可能等于 0，直接结束循环。\
3.双指针搜索：在 i 之后的区间 [i+1, n-1] 中设置两个指针：left 指向区间首部，right 指向区间尾部。\
&emsp;&emsp;计算 total = nums[i] + nums[left] + nums[right]。\
&emsp;&emsp;若 total < 0：说明和太小，left 右移。\
&emsp;&emsp;若 total > 0：说明和太大，right 左移。\
&emsp;&emsp;若 total == 0：找到一组解！\
&emsp;&emsp;&emsp;&emsp;去重逻辑：继续移动 left 和 right，跳过所有与刚才相同的值，防止结果集中出现重复的三元组。\
4.返回结果：收集所有满足条件的三元组。
## python3代码实现
```python
from typing import List

class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        n = len(nums)
        res = []
        # 1. 必须先排序
        nums.sort()
        
        for i in range(n):
            # 2. 剪枝优化：如果当前数大于0，后面三个数之和必然大于0
            if nums[i] > 0:
                break
            
            # 3. 去重：如果当前数和上一个数相同，跳过
            if i > 0 and nums[i] == nums[i-1]:
                continue
            
            # 4. 双指针过程
            left = i + 1
            right = n - 1
            
            while left < right:
                total = nums[i] + nums[left] + nums[right]
                
                if total < 0:
                    left += 1
                elif total > 0:
                    right -= 1
                else:
                    # 找到一组解
                    res.append([nums[i], nums[left], nums[right]])
                    
                    # 5. 去重：跳过左指针和右指针重复的值
                    while left < right and nums[left] == nums[left + 1]:
                        left += 1
                    while left < right and nums[right] == nums[right - 1]:
                        right -= 1
                    
                    # 指针同时收缩
                    left += 1
                    right -= 1
                    
        return res

# --- 测试代码 ---
if __name__ == "__main__":
    sol = Solution()
    
    # 测试用例 1
    case1 = [-1, 0, 1, 2, -1, -4]
    print(f"输入 1: {case1} -> 输出: {sol.threeSum(case1)}") 
    # 预期: [[-1, -1, 2], [-1, 0, 1]]
```
## 逻辑图解（以 [-1, 0, 1, 2, -1, -4] 为例）
1.排序：得到 [-4, -1, -1, 0, 1, 2]。\
2.i=0 (nums[i]=-4)：\
&emsp;&emsp;left 指向 -1，right 指向 2。\
&emsp;&emsp;-4 + (-1) + 2 = -3 < 0，left 右移。\
&emsp;&emsp;一路找下去，没有和为 0 的情况。\
3.i=1 (nums[i]=-1)：\
&emsp;&emsp;left 指向下一个 -1，right 指向 2。\
&emsp;&emsp;-1 + (-1) + 2 = 0。记录 [-1, -1, 2]。\
&emsp;&emsp;跳过重复，left 移动到 0，right 移动到 1。\
&emsp;&emsp;-1 + 0 + 1 = 0。记录 [-1, 0, 1]。\
4.i=2 (nums[i]=-1)：\
&emsp;&emsp;因为 nums[2] == nums[1]，触发去重逻辑，跳过。\
5.后续过程：直到结束。
## 复杂度分析
时间复杂度: $O(n^2)$ \
&emsp;&emsp;数组排序需要 $O(nlogn)$。\
&emsp;&emsp;外层 for 循环执行 n次，内层双指针扫描执行 n 次，总共 $O(n^2)%。\
空间复杂度: $O(logn)$ 或 $O(n)$\
&emsp;&emsp;主要取决于排序算法（如 Python 的 Timsort）所需的辅助空间，不计入存储结果集的空间。


# 42.接雨水
给定 n 个非负整数表示每个宽度为 1 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。
这是一个经典的“木桶原理”应用问题，也是双指针算法的巅峰之作。
问题的本质是：动态维护左右两端的“最高瓶颈”。
对于位置 i，它能接的水量为：min(左边最高,右边最高)−当前高度。
通过使用从两端向中间汇合的双指针，我们可以实时更新左侧和右侧观察到的最大高度，并始终处理高度较低的那一端，因为低的那一端决定了“水面的上限”（即木桶效应）。最优解法是使用双指针法（一次遍历， $O(1)$ 空间）。
## 算法思路（双指针递进）
1.初始化：\
&emsp;&emsp;左指针 left = 0，右指针 right = n - 1。\
&emsp;&emsp;记录左侧已知的最高高度 left_max = 0，右侧已知的最高高度 right_max = 0。\
2.收缩过程：\
&emsp;&emsp;当 left < right 时执行循环：\
&emsp;&emsp;更新极值：更新 left_max 为 height[left] 的最大值，更新 right_max 为 height[right] 的最大值。\
&emsp;&emsp;比较并计算：\
&emsp;&emsp;&emsp;如果 height[left] < height[right]：\
&emsp;&emsp;&emsp;&emsp;由于左侧较低，水面高度由 left_max 决定。\
&emsp;&emsp;&emsp;&emsp;当前 left 位置接水量 = left_max - height[left]。\
&emsp;&emsp;&emsp;&emsp;left 指针右移。\
&emsp;&emsp;&emsp;否则（右侧较低或相等）：\
&emsp;&emsp;&emsp;&emsp;水面高度由 right_max 决定。\
&emsp;&emsp;&emsp;&emsp;当前 right 位置接水量 = right_max - height[right]。\
&emsp;&emsp;&emsp;&emsp;right 指针左移。\
3.返回结果：累加所有位置的接水量即为总和。
## python3代码实现
```python
from typing import List

class Solution:
    def trap(self, height: List[int]) -> int:
        if not height:
            return 0
            
        left, right = 0, len(height) - 1
        left_max, right_max = 0, 0
        res = 0
        
        while left < right:
            # 实时更新左右两边的最高纪录
            left_max = max(left_max, height[left])
            right_max = max(right_max, height[right])
            
            # 核心逻辑：哪边低，就先结算哪边
            # 因为低的那边已经遇到了它这一侧的“天花板”，而另一侧肯定有更高的挡着
            if height[left] < height[right]:
                res += left_max - height[left]
                left += 1
            else:
                res += right_max - height[right]
                right -= 1
                
        return res

# --- 测试代码 ---
if __name__ == "__main__":
    sol = Solution()
    
    # 测试用例 1
    case1 = [0,1,0,2,1,0,1,3,2,1,2,1]
    print(f"输入 1: {case1} -> 输出: {sol.trap(case1)}") # 预期: 6
    
    # 测试用例 2
    case2 = [4,2,0,3,2,5]
    print(f"输入 2: {case2} -> 输出: {sol.trap(case2)}") # 预期: 9
```
## 逻辑图解（以 [4, 2, 0, 3, 2, 5] 为例）
1.初始：L=0(h=4), R=5(h=5)。L_max=4, R_max=5。\
2.计算：h[L] < h[R]，处理左边。\
&emsp;&emsp;res += 4 - 4 = 0。L 移到 1。\
3.下步：L=1(h=2), R=5(h=5)。L_max=4, R_max=5。\
&emsp;&emsp;res += 4 - 2 = 2。L 移到 2。\
4.下步：L=2(h=0), R=5(h=5)。L_max=4, R_max=5。\
&emsp;&emsp;res += 4 - 0 = 4（累计6）。L 移到 3。\
5.下步：L=3(h=3), R=5(h=5)。L_max=4, R_max=5。\
&emsp;&emsp;res += 4 - 3 = 1（累计7）。L 移到 4。\
6.下步：L=4(h=2), R=5(h=5)。L_max=4, R_max=5。\
&emsp;&emsp;res += 4 - 2 = 2（累计9）。L 移到 5。\
7.结束：指针相遇，返回 9
## 复杂度分析
时间复杂度: $O(n)$\
两个指针共同遍历数组一次，每个元素仅被访问一次。\
空间复杂度: $O(1)$\
只使用了 left, right, left_max, right_max 等常数个变量。相比于使用动态规划（需要两个数组存储左右最高值，空间 $O(n)$），双指针法达到了空间最优

# 3.无重复字符的最长字串
给定一个字符串 s ，请你找出其中不含有重复字符的 最长 子串 的长度。
这是一个经典的“滑动窗口（Sliding Window）”变长区间问题。
问题的本质是：动态维护一个没有重复元素的“活动窗口”。
我们使用两个指针表示窗口的左右边界。右指针不断向右扩张以纳入新字符；一旦新加入的字符与窗口内既有字符重复，左指针必须立即跳过重复字符出现的位置，从而保证窗口内的唯一性。
最优解法是使用哈希表（Hash Map）记录索引的滑动窗口法。
## 算法思路（双指针 + 哈希表优化）
1.初始化：\
&emsp;&emsp;左指针 left：指向当前无重复子串的起始位置，初始为 0。\
&emsp;&emsp;哈希表 char_map：用于存储字符及其最近一次出现的索引位置。格式为 {字符: 索引}。\
&emsp;&emsp;最大长度 max_len：记录扫描过程中的最优结果。\
2.右指针遍历：\
&emsp;&emsp;使用变量 right 遍历字符串 s。\
&emsp;&emsp;检查重复：如果 s[right] 已经在 char_map 中，说明遇到了重复字符。\
&emsp;&emsp;&emsp;&emsp;移动左边界：将 left 更新为 max(left, char_map[s[right]] + 1)。\
&emsp;&emsp;&emsp;&emsp;注：使用 max 是为了防止 left 回跳到已经失效的旧位置。\
3.更新状态：\
&emsp;&emsp;更新 char_map[s[right]] 为当前的最新索引 right。\
&emsp;&emsp;计算当前窗口长度 right - left + 1，并更新 max_len。\
4.返回结果：遍历结束，返回 max_len。
## pthon3代码实现
```python
class Solution:
    def lengthOfLongestSubstring(self, s: str) -> int:
        # 1. 建立哈希表记录字符最后出现的位置
        char_map = {}
        left = 0
        max_len = 0
        
        # 2. right 指针不断向右移动
        for right in range(len(s)):
            # 3. 如果当前字符已经在窗口中出现过
            if s[right] in char_map:
                # 4. 核心：左指针跳到重复字符上次出现位置的下一个
                # 必须用 max 确保 left 不会往回走
                left = max(left, char_map[s[right]] + 1)
            
            # 5. 更新当前字符的最新位置
            char_map[s[right]] = right
            
            # 6. 更新全局最大长度
            max_len = max(max_len, right - left + 1)
            
        return max_len

# --- 测试代码 ---
if __name__ == "__main__":
    sol = Solution()
    
    # 测试用例 1
    case1 = "abcabcbb"
    print(f"输入 1: {case1} -> 输出: {sol.lengthOfLongestSubstring(case1)}") # 预期: 3 ("abc")
    
    # 测试用例 2
    case2 = "pwwkew"
    print(f"输入 2: {case2} -> 输出: {sol.lengthOfLongestSubstring(case2)}") # 预期: 3 ("wke")
```
## 逻辑图解（以 "pwwkew" 为例）
1.初始：L=0, R=0, s[R]='p'。char_map={'p':0}, max_len=1。\
2.下一步：R=1, s[R]='w'。char_map={'p':0, 'w':1}, max_len=2。\
3.下一步：R=2, s[R]='w'。重复！\
&emsp;&emsp;L 跳到 char_map['w']+1 = 2。\
&emsp;&emsp;char_map['w'] 更新为 2。\
&emsp;&emsp;max_len 仍为 2。窗口现在是 s[2...2] 即 "w"。\
4.下一步：R=3, s[R]='k'。char_map={'p':0, 'w':2, 'k':3}, max_len=2。窗口 "wk"。\
5.下一步：R=4, s[R]='e'。char_map={'p':0, 'w':2, 'k':3, 'e':4}, max_len=3。窗口 "wke"。\
6.下一步：R=5, s[R]='w'。重复！\
&emsp;&emsp;L 跳到 char_map['w']+1 = 3。\
&emsp;&emsp;窗口变为 "kew"，max_len 保持 3。

# 438.找到字符串中左右字母异位词
给定两个字符串 s 和 p，找到 s 中所有 p 的 异位词 的子串，返回这些子串的起始索引。不考虑答案输出的顺序。这是一个经典的“定长滑动窗口（Fixed-size Sliding Window）”字符频率匹配问题。
问题的本质是：维护两个“字符频率计数器”的实时同步。
我们利用数组或哈希表统计 p中每个字符出现的次数。随后，在 s上滑动一个大小固定的窗口，每向右移动一步，就更新窗口内新增和移除字符的频率。如果窗口内的频率分布与 p的分布完全一致，则找到了一个异位词。
最优解法是使用 26 位数组记录频率的滑动窗口法。
## 算法思路（定长滑动窗口 + 数组计数）
1.特殊情况处理：如果 len(p)>len(s)，显然不可能存在异位词，直接返回空列表。\
2.初始化频率表：\
&emsp;&emsp;创建两个长度为 26 的数组 p_count 和 s_count（对应 a-z）。\
&emsp;&emsp;先统计字符串 p的频率，同时统计 s 中前 len(p)个字符的频率。\
3.首位判断：对比 p_count 和 s_count 是否相等。若相等，将索引 0 加入结果集。\
4.窗口滑动：\
&emsp;&emsp;从索引 i = 0 开始向右滑动窗口。\
&emsp;&emsp;移除左侧：将 s[i]移出窗口，s_count 对应位置减 1。\
&emsp;&emsp;加入右侧：将 s[i+len(p)]移入窗口，s_count 对应位置加 1。\
&emsp;&emsp;对比频率：每次滑动后，直接比较两个数组。若 s_count == p_count，则将起始索引 i + 1 加入结果。\
5.返回结果：遍历结束，返回所有记录的索引。
## python3代码实现
```python
from typing import List

class Solution:
    def findAnagrams(self, s: str, p: str) -> List[int]:
        n, m = len(s), len(p)
        if m > n:
            return []
            
        res = []
        # 1. 初始化两个频率数组 (a-z 映射到 0-25)
        p_count = [0] * 26
        s_count = [0] * 26
        
        # 2. 统计 p 的频率和 s 第一个窗口的频率
        for i in range(m):
            p_count[ord(p[i]) - ord('a')] += 1
            s_count[ord(s[i]) - ord('a')] += 1
            
        # 3. 检查第一个窗口
        if s_count == p_count:
            res.append(0)
            
        # 4. 开始滑动窗口
        # i 代表旧窗口的起始位置，即将被移出的元素
        for i in range(n - m):
            # 移除左边字符
            s_count[ord(s[i]) - ord('a')] -= 1
            # 增加右边字符
            s_count[ord(s[i + m]) - ord('a')] += 1
            
            # 直接比较数组（Python 列表比较是 O(26) 的常数级操作）
            if s_count == p_count:
                res.append(i + 1)
                
        return res

# --- 测试代码 ---
if __name__ == "__main__":
    sol = Solution()
    
    # 测试用例 1
    s1, p1 = "cbaebabacd", "abc"
    print(f"输入: s='{s1}', p='{p1}' -> 输出: {sol.findAnagrams(s1, p1)}") 
    # 预期: [0, 6] (分别是 "cba" 和 "bac")
    
    # 测试用例 2
    s2, p2 = "abab", "ab"
    print(f"输入: s='{s2}', p='{p2}' -> 输出: {sol.findAnagrams(s2, p2)}") 
    # 预期: [0, 1, 2]
```
## 逻辑图解（以 s="cbaebabacd", p="abc" 为例）
1.初始状态：窗口 s[0...2] 为 "cba"。\
&emsp;&emsp;p_count: {a:1, b:1, c:1}, s_count: {a:1, b:1, c:1}。\
&emsp;&emsp;匹配成功，记下 0。\
2.滑动 1 步：移出 'c'，移入 'e'。窗口变为 "bae"。\
&emsp;&emsp;s_count: {a:1, b:1, e:1}。\
&emsp;&emsp;匹配失败。\
3.持续滑动：直到窗口移入最后一段 "bac"。\
&emsp;&emsp;此时左侧移出 'a'，右侧移入 'c'。\
&emsp;&emsp;s_count 重新变回 {a:1, b:1, c:1}。\
&emsp;&emsp;匹配成功，记下起始索引。
## 复杂度分析
时间复杂度：O(n+m)（或更精确地说是 O(n+m×Σ)，Σ是字符集大小 26）。\
&emsp;&emsp;初始化计数需要 O(m)。\
&emsp;&emsp;滑动窗口遍历 s 需要 O(n)，每次滑动中数组的比较和更新都是 O(26)的常数操作。\
空间复杂度：O(1)。\
&emsp;&emsp;虽然使用了 p_count 和 s_count 数组，但它们的大小固定为 26，不随输入字符串的长度而增长，属于常数空间开销。
