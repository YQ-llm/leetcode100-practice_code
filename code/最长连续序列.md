# 128.最长连续序列
给定一个未排序的整数数组 nums ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。
请你设计并实现时间复杂度为 O(n) 的算法解决此问题。\
这是一个经典的哈希表（Hash Set）查找与逻辑去重问题。最优解法是使用哈希集合（Hash Set）进行线性探测。
## 算法思路（哈希集合 + 序列起点探测）
1.构建去重雷达：将原数组 数字 转换为一个哈希集合 数字集。这一步可以实现 $O(1)$ 的元素查找，并自动处理重复数字。
2.寻找序列起点（领头羊）：遍历集合中的每一个数字 在：\
&emsp;&emsp;检查数字 - 1 是否存在于集合中。\
&emsp;&emsp;如果 不存在：说明 在 是一条连续序列的起点。\
&emsp;&emsp;如果存在：说明在 只是某个序列的中间成员，直接跳过，不做处理。\
3.顺藤摸瓜计数：一旦确认了起点 在，就从该起点开始，不断探测 数字 + 1，数字 + 2, ... 直到数字中断。\
4.动态更新最大长度：记录当前探测到的序列长度，并与全局最大长度 最大长度 进行比对更新。
## Python3代码实现
```python
from typing import List

class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        # 1. 转换为哈希集合，实现 O(1) 查找
        num_set = set(nums)
        max_len = 0
        
        # 2. 遍历集合
        for num in num_set:
            # 3. 核心判断：只有当 num 是一个序列的起点时才开始计数
            # 如果 num - 1 在集合中，说明当前 num 不是起点，直接跳过
            if (num - 1) not in num_set:
                current_num = num
                current_len = 1
                
                # 4. 从起点向后持续探测连续数字
                while (current_num + 1) in num_set:
                    current_num += 1
                    current_len += 1
                
                # 5. 更新全局最大长度
                max_len = max(max_len, current_len)
                
        return max_len

# --- 测试代码 ---
if __name__ == "__main__":
    sol = Solution()
    
    # 测试用例 1: 序列为 [1, 2, 3, 4], [100], [200]
    nums1 = [100, 4, 200, 1, 3, 2]
    print(f"输入: {nums1} -> 输出: {sol.longestConsecutive(nums1)}") # 预期: 4
    
    # 测试用例 2: 序列为 [0, 1, 2, 3, 4, 5, 6, 7, 8]
    nums2 = [0, 3, 7, 2, 5, 8, 4, 6, 0, 1]
    print(f"输入: {nums2} -> 输出: {sol.longestConsecutive(nums2)}") # 预期: 9
```
## 逻辑图解（以 $nums = [100, 4, 200, 1, 3, 2]$ 为例）
1.Set 建立：num_set = {1, 2, 3, 4, 100, 200}。\
2.迭代过程：\
&emsp;&emsp;遇到100：99 不在，是起点。向后找 101，不在。长度为 1。max_len = 1。\
&emsp;&emsp;遇到4：3 在 Set 中，说明 4 不是起点，直接跳过。\
&emsp;&emsp;遇到200：199 不在，是起点。向后找 201，不在。长度为 1。最大长度 仍为 1。\
&emsp;&emsp;遇到1：0 不在，是起点。向后找 2, 3, 4，都在！长度达到 4。max_len = 4。\
&emsp;&emsp;遇到3、2：它们的前驱数字都在 Set 中，全部跳过。\
3.结论：返回 4。
## 复杂度分析
时间复杂度: $O(n)$\
&emsp;&emsp;虽然代码中存在 为了嵌套尽管 的结构，但只有当一个数字是“起点”时才会进入 尽管。\
&emsp;&emsp;数组中的每个数字在整个过程中最多被访问两次（一次在 为了 中作为判断，一次在 尽管 中作为序列增长）。因此，总操作步数与 
$n$ 成正比。\
空间复杂度: $O(n)$\
&emsp;&emsp;我们需要一个哈希集合来存储所有的数字，空间消耗与输入数组的长度呈线性关系
